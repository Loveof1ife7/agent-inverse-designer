# Core Output Schema

The `auto_generate_4x4x4.py` pipeline writes one run directory per group.

Expected files:

- `constraints_<group>.json`
  Symmetry constraints exported by `constraints_solver.py`.
- `<group>-architecture.csv`
  Raw generated dataset in tabular form.
- `abaqus_txt/*.txt`
  One reconstructed Abaqus-style structure file per accepted sample.
- `crystal_4x4x4/*.txt`
  One expanded crystal structure file per accepted sample.
- `crystal_4x4x4/errors.log`
  Crystal expansion failures, if any.
- `summary.json`
  Run-level summary generated by the pipeline.

Scheduler-level postprocessing also writes:

- `generated_data_manifest.json`
  Structured manifest for downstream indexing.
- `knowledge_base_seed.jsonl`
  One JSON record per generated sample, suitable for building a knowledge base.

Recommended knowledge-base ingestion unit:

- one line in `knowledge_base_seed.jsonl`
- keyed by `structure_id`
- linked to:
  - `csv_path`
  - `abaqus_txt_path`
  - `crystal_txt_path`
  - `constraints_path`
